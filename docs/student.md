

这份技术文档是为 ** 负责人3（学生能力建模与大模型工程负责人）**设计的。该模块的核心目标是将简历（PDF/Word）及用户录入的非结构化文本，通过 LLM 转化为系统可计算的结构化能力向量 。

----------

## 学生能力建模模块（Student Profiling Module）技术文档

### 1. 模块目标

-   **非结构化提取**：利用大模型从简历文本中精准抽取技能、项目、经历等关键信息 。
    
-   **能力标准化**：对抽取的原始标签进行归一化处理（如：将“熟练使用 SpringBoot”映射为技能字段并对熟练度评分） 。
    
-   **向量化输出**：生成符合负责人 1 定义的标准化能力向量结构，供匹配引擎调用 。
    

### 2. 核心数据流程

1.  **文本预处理**：使用 `PyMuPDF` 或 `docx2txt` 提取简历内容，去除乱码。
    
2.  **LLM 抽取（LangChain）**：采用 `Structured Output` 模式，强制模型输出固定 Schema 的 JSON 。
    
3.  **能力评分与量化**：基于项目描述的深度和技术栈的匹配度，由大语言模型评判职业素养评分，发展潜力评分。
    

----------

### 4. 技术实现方案

#### 4.1 简历解析 Prompt 策略

为确保输出稳定性（严格的json格式和字段约束），参考以下提示词结构：
```
你是专业的简历信息提取分析师，需要从以下简历文本中精准提取指定信息，并严格按照要求的JSON格式输出，不得添加任何额外说明、注释或无关字段，评分必须为1-5的整数，evidence字段需准确引用简历原文中的对应表述。

【提取字段要求】
1. skills：数组类型，每个元素包含name（技能名称）、level（熟练度评分，1=入门，5=精通）、evidence（简历中支撑该技能的原文）；
2. certificates：数组类型，每个元素包含name（证书名称）、evidence（简历中支撑该证书的原文）；
3. Experience：数组类型，每个元素包含name（项目/实习经历名称）、evidence（简历中支撑该经历的原文）；
4. Professionalism：对象类型，包含level（职业素养评分，1=最低，5=最高），基于简历中体现的责任心、执行力、团队协作等维度评估；
5. Potential：对象类型，包含level（发展潜力评分，1=最低，5=最高），基于简历中体现的学习能力、成长空间、技术沉淀等维度评估。

【输出要求】
- 严格遵循JSON格式，字段名与上述要求完全一致（大小写敏感）；
- 无对应信息的字段填充空数组/合理默认值（如评分字段无信息则填1）；
- 禁止输出JSON以外的任何内容（包括但不限于“以下是提取结果：”等话术）；
- evidence字段必须是简历原文的精准截取，不得篡改或概括。
```
